# 电商比价助手

一个面向大促场景的浏览器扩展，聚焦同一款竞品在不同电商平台之间的价格对比与监控。

当前版本重点服务于官方店竞品监控场景，已支持京东、淘宝、天猫商品页识别，支持手动绑定同款、关注清单、价格历史记录，以及浏览器开启期间的低频自动刷新。

## 适用场景

- 大促期间跟踪同一款商品在京东和天猫官方店的价格变化
- 运营、投放、竞品分析时快速对比不同平台同款价格
- 手动维护一批重点 SKU 的跨平台价格监控清单
- 在本地积累历史价格记录，观察临时调价、先涨后降等行为

## 当前能力

- 支持京东商品页价格提取
- 支持淘宝 / 天猫商品页价格提取
- 支持从当前商品页快速搜索另一平台同款
- 支持从当前商品页搜索同品牌 / 同店铺结果
- 支持把不同平台页面手动绑定到同一个监控项
- 支持关注清单与监控状态展示
- 支持价格历史记录与图表展示
- 支持浏览器开启期间的后台自动刷新
- 所有数据保存在浏览器本地，不依赖后端服务

## 当前支持平台

- 京东
- 淘宝
- 天猫

## 功能说明

### 1. 商品页识别

打开京东、淘宝或天猫商品页后，扩展会尝试自动识别：

- 商品标题
- 当前价格
- 划线价
- 店铺名称
- 商品图片

如果页面没有自动识别成功，也可以通过侧边栏中的“手动提取当前页商品”重新触发。

### 2. 跨平台比价

在当前商品识别成功后，可以直接：

- 搜索另一平台同款商品
- 搜索另一平台同品牌 / 同店铺商品
- 把当前商品加入关注清单

### 3. 关注清单与手动配对

这个项目当前采用“手动绑定同款”的方式来确保准确率。

推荐使用流程：

1. 在京东商品页打开侧边栏并识别商品
2. 点击“加入清单”
3. 用“搜商品”或“搜同店”打开淘宝 / 天猫对应页面
4. 在目标页面回到清单页，点击“绑定当前页”
5. 完成后，一个监控项里就会同时记录京东和淘宝 / 天猫的价格

这套方式更适合大促期间针对重点竞品做小批量、准确优先的监控。

### 4. 自动刷新

关注清单页支持自动刷新：

- 可开启 / 关闭自动刷新
- 可选择刷新间隔：30 / 60 / 120 / 240 分钟
- 可手动点击“立即刷新”
- 会显示最近一次刷新时间和执行结果

自动刷新机制说明：

- 依赖浏览器扩展后台能力，不是云端任务
- 只有在浏览器保持开启时才会运行
- 会按清单顺序低频刷新，尽量降低风险
- 刷新过程中不会覆盖你当前正在查看的商品卡片状态

### 5. 历史记录

每个监控项会保留价格历史，可在历史页查看：

- 时间维度价格变化
- 不同平台的最近价格
- 最低价 / 最高价
- 最近记录列表

## 安装方式

### 方式一：直接加载源码

1. 下载或克隆本仓库
2. 打开 Chrome 或 Edge 的扩展管理页
3. 开启“开发者模式”
4. 点击“加载已解压的扩展程序”
5. 选择当前项目目录

### 方式二：从 GitHub 克隆

```bash
git clone <your-repo-url>
cd price-compare-extension
```

然后按“方式一”加载到浏览器即可。

## 使用流程

1. 打开京东 / 淘宝 / 天猫商品页
2. 打开扩展侧边栏
3. 确认商品已识别成功
4. 将商品加入关注清单
5. 打开另一平台的同款商品页
6. 在清单中把当前页绑定到对应监控项
7. 开启自动刷新或手动刷新
8. 在历史页查看价格变化

## 项目结构

```text
background/   后台 Service Worker，负责消息分发、清单同步、自动刷新
content/      京东 / 淘宝 / 天猫页面内容提取逻辑
sidepanel/    扩展侧边栏 UI
utils/        平台识别、存储、通用工具
icons/        扩展图标
manifest.json Chrome Extension Manifest V3 配置
```

## 权限说明

本项目目前使用以下浏览器权限：

- `activeTab`
  用于读取当前活动标签页中的商品信息
- `tabs`
  用于打开搜索结果页、后台创建刷新标签页
- `storage`
  用于保存关注清单、价格历史、刷新设置
- `sidePanel`
  用于使用浏览器侧边栏作为主要交互界面
- `alarms`
  用于定时触发自动刷新任务

Host 权限仅限：

- `*.jd.com`
- `*.taobao.com`
- `*.tmall.com`

## 数据与隐私

- 本项目当前不依赖后端服务
- 监控清单、历史记录、设置等数据都保存在浏览器本地存储中
- 默认不会上传你的商品数据到外部服务器

如果后续引入服务端能力，建议单独在 README 和隐私说明中补充说明。

## 当前限制

- 目前更适合手动维护重点竞品，而不是大规模自动化抓取
- “同款商品”仍然依赖人工确认和绑定
- 自动刷新不是 24/7 云端监控，浏览器关闭后不会继续执行
- 商品页结构变化后，页面提取逻辑可能需要跟进适配
- 当前重点优化的是京东与天猫官方店竞品监控场景
